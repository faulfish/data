Release Notes:

OmniDiskSweeper for Mac

Release Notes: What's new, updated, or fixed

Version 1.9 — Monday, Dec 30, 2013

  • New
  • Updated
  • Fixed
  • More Responsive — OmniDiskSweeper is much more responsive when scanning a slow disk. (The interface used to block while scanning the list of files contained in a folder, which could cause very long pauses when scanning slow disks.)
  • Faster Scans of Remote Disks — Switched to scanning directory contents using API that only collects the small set of attributes that OmniDiskSweeper is actually going to use, so it can scan remote disks much more quickly.
  • Smarter Scan Order — Rather than scanning a directory to completion before starting on any other directories, OmniDiskSweeper will automatically skip between folders so you can more quickly get a sense of how big each directory is relative to the others.
  • System Requirements — The minimum system requirement is now 10.8 (Mountain Lion).

Version 1.8 — Monday, July 23, 2012

  • New
  • Updated
  • Fixed
  • 64-bit — OmniDiskSweeper is now a 64-bit app, which means it can handle much larger sets of files.
  • Gatekeeper — The app is now signed with Developer ID so it works well with Gatekeeper on Mountain Lion.
  • System Requirements — The minimum system requirement is now 10.6.8 (Snow Leopard).
  • Decimal Sizes — File sizes are now reported using standard metric prefixes (1 kB = 10^3 bytes = 1,000 bytes; 1 MB = 10^6 bytes = 1,000,000 bytes) rather than using binary-based prefixes (1 kB = 2^10 bytes = 1,024 bytes; 1 MB = 2^20 bytes = 1,048,576 bytes).
  • Interface — OmniDiskSweeper has a new app icon!
  • Lion Automounts — Lion’s automount folders are no longer displayed in the network drive list.
